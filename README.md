# Documentação do Desafio

Este repositório contém a documentação e os artefatos produzidos durante a execução do desafio de testes.

## 📌 Análise Inicial da Aplicação

Durante a análise inicial da aplicação, foram avaliadas as principais funcionalidades, fluxos disponíveis e possíveis pontos críticos do sistema.
Algumas perguntas foram levantadas durante o processo de análise. Abaixo estão as perguntas e suas respectivas respostas:

### Qual é o objetivo da aplicação?
- O objetivo da aplicação é fornecer uma **plataforma online para transmissão de cursos**, permitindo que usuários criem e disponibilizem conteúdos educacionais.

### Quais são os principais fluxos disponíveis?
- **Fluxo de cadastro de cursos**
- **Fluxo de listagem de cursos**

### Quais pontos do sistema são mais críticos para testes?
- O fluxo considerado mais crítico é o **cadastramento de cursos**, pois é uma funcionalidade essencial para o funcionamento da plataforma. Caso esse fluxo apresente falhas, novos cursos não poderão ser cadastrados no sistema.

## 🧠 Decisões para Criação dos Testes

Para a criação dos testes, se considerou a principal funcionalidade (criação de curso) do sistema e os cenários mais criticos para o usuário. A partir da exploração do site, foram definidos cenários de testes positivos e negativos, validação de campos e possiveis falhas esperadas.

## 🔎 Raciocínio Durante a Análise
Para a análise, primeiro foi realizado a técnica de ad hoc(uso do meu conhecimento previo em formulários) e técnica de testes exploratórios para entendimento da funcionalidade proposta. Em seguida, foram identificadas as principais ações que o usuário poderia realizar no sistema. Então elaborado cenários de teste considerando tanto os comportamentos esperados quanto possíveis falhas, e após realização dos casos de teste focando somente nos cenários de teste que passaram e não apresentaram falhas para uma documentação de comportamento correta
